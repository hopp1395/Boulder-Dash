# Prüfstand 13 — Unregelmäßig geformte Cave (kein BD1-Original, nur über den Testmodus F5 erreichbar).
#
# Zu prüfen: Eine Cave muss weder rechteckig sein noch 40x22 groß. Width/Height spannen nur das
# GITTER auf; die Höhle darin ist ein Achteck mit einem Tunnel in eine Ausgangskammer. Alles um sie
# herum ist Void ('_') — außerhalb der Cave wird nichts gezeichnet, dort bleibt es schwarz statt der
# früheren Stahl-Randmauer. Auch beim Auf- und Zudecken: Es geht die SILHOUETTE der Höhle auf und zu,
# nicht das Rechteck des Gitters.
#
# Die Karte muss das Gitter nicht ausfüllen: Zeilen dürfen kürzer sein (der Rest ist Void), und die
# drei untersten Zeilen fehlen ganz. Eine Zeile aus lauter Void braucht ein '_', weil der Parser
# Leerzeilen überliest — siehe die erste Kartenzeile.
#
# Der Stahlring muss trotzdem lückenlos sein; darauf besteht der Parser (CaveTextFile), denn auf ihm
# beruht die ganze Physik: Kein Objekt greift je über den Gitterrand hinaus.
